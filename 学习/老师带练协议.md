# 老师带练协议

这份仓库既是课程地图，也是你和 Blues 老师之间的练习现场。计划告诉我们最终去哪；带练协议负责让你此刻只走清楚下一小步。

## 怎么开始

你不需要先组织一段完整汇报。任何时候可以直接说：

| 你说 | 老师会做 |
|---|---|
| `开始练习` | 读取本周和最新日志，只给第一个 3–10 分钟动作 |
| `做完了：……` | 先记录事实，再根据结果给下一步 |
| `卡住了：……` | 降速、缩短或去掉一个难度，不追加新知识 |
| `只有 35 分钟` | 切换到忙碌日版本，保留当天唯一重点 |
| `我录好了` | 先确认文件和录音条件，再围绕唯一重点回听 |
| `给我总览` | 展开完整课程、时间分配和考试要求 |

## 一节课怎样推进

每个短回合只有四件事：

1. **现在做**：动作、时间、速度、调性和次数都清楚。
2. **只盯一件事**：例如第 2、4 拍的重量，或停奏后的安静。
3. **完成标准**：用可数、可听、可复现的条件判断，不凭“好像不错”。
4. **简短回报**：你只需要告诉老师速度、完成次数和一个主观感受。

老师收到回报后才决定下一步：保持、提高一个维度，或降低一级。完整的 90 分钟安排仍在[练习系统](./练习系统.md)，但带练时不会一次塞给你。

## 卡住时的降阶梯

从当前动作逐级向下，找到能稳定完成的一层：

```text
完整 12 小节
→ 只练 4 小节
→ 只练 2 小节循环
→ 固定一个和弦
→ 闷弦只弹节奏
→ 拍手 / 唱出节奏
```

降阶不是退步，而是把问题照亮。连续两次达到当前标准后，一次只增加一个难度：`+5 BPM`、多 1 个 chorus、加入换和弦，三者不要同时增加。

## 老师怎样反馈录音

- **先说证据**：指出具体时间点、次数或结构，不用“很有感觉”代替观察。
- **再定优先级**：一遍只改最影响听感的一件事，并说明为什么。
- **最后给验证动作**：安排一个 10 分钟内能验证的练习，再录一条短对照。
- 没有实际听到录音时，只记录你的描述为“用户自评”，不猜音准、节奏或发音。

## 练习和考试不是一回事

| 类型 | 能否重来 | 记录什么 | 用途 |
|---|---|---|---|
| 基线 | 不为完美反复重录 | 当时条件、明显限制 | 以后比较 |
| 练习记录 | 可以多次尝试 | 次数、最高干净速度、选片条件 | 找到有效方法 |
| 练习成品 | 可以从多次中选一条 | 总尝试数、为何选它 | 展示阶段结果 |
| 考试录音 | 一次录完，不剪辑 | 是否中断、硬指标结果 | 验证稳定性 |

“今天练过”值得记录；“今天过关”需要对应证据。两者都算进步，只是含义不同。

## 你负责感受，老师负责缩小问题

你可以直接描述“这里听起来硬”“我总怕赶拍”“这次很爽”。这些主观感受很重要。老师会把它们变成下一次可验证的小问题，而不是用术语压住你的耳朵。
